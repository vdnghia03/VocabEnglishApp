# 2.1 Cấu trúc Dữ liệu (Data Structure)

## 1. Định dạng lưu trữ
Dữ liệu được lưu trữ dưới dạng **JavaScript Modules** (`.js`), sử dụng cú pháp `export const` để có thể import trực tiếp vào ứng dụng mà không cần parsing JSON.

## 2. Lesson Object (Đối tượng Bài học)
Mỗi bài học (Lesson) là một Object chứa thông tin tổng quan và danh sách từ vựng.

### Các trường thông tin (Fields):
| Tên trường | Kiểu dữ liệu | Mô tả | Ví dụ |
| :--- | :--- | :--- | :--- |
| `id` | Number | Mã định danh duy nhất của bài học. | `1` |
| `title` | String | Tiêu đề hiển thị (bao gồm số bài và tên bài). | `"Bài 1: Civilian - Vacant"` |
| `description` | String | Mô tả ngắn gọn về nội dung/chủ đề bài học. | `"20 từ vựng nâng cao về..."` |
| `words` | Array | Mảng chứa danh sách các đối tượng từ vựng. | `[...]` |

## 3. Word Object (Đối tượng Từ vựng)
Nằm bên trong mảng `words` của Lesson Object.

### Các trường thông tin (Fields):
| Tên trường | Kiểu dữ liệu | Mô tả | Quy tắc |
| :--- | :--- | :--- | :--- |
| `id` | Number | Số thứ tự của từ trong bài. | Tăng dần từ 1 đến 20. |
| `vocab` | String | Từ vựng tiếng Anh (gốc). | Viết thường toàn bộ (trừ tên riêng). |
| `meaning` | String | Nghĩa tiếng Việt. | Tối đa 2 nghĩa chính, ngắn gọn. |

## 4. Mẫu code chuẩn (Code Snippet)
Dưới đây là mẫu chuẩn cho một file dữ liệu (ví dụ `lesson1.js`):

```javascript
export const lesson1 = {
  id: 1,
  title: "Bài 1: Civilian - Vacant",
  description: "20 từ vựng nâng cao về con người và trạng thái.",
  words: [
    { id: 1, vocab: "civilian", meaning: "thường dân" },
    { id: 2, vocab: "complicated", meaning: "phức tạp, rắc rối" },
    // ... các từ tiếp theo
  ]
};
```

## 5. Quy ước đặt tên (Naming Conventions)

Tên file: lesson{id}.js (Ví dụ: lesson1.js, lesson10.js).

Tên biến xuất ra: lesson{id} (Ví dụ: export const lesson1 = ...).

Xử lý văn bản: - Đảm bảo tiếng Việt có dấu chuẩn Unicode.

Sửa các lỗi hiển thị do font cũ (ví dụ: ngƣời -> người).
