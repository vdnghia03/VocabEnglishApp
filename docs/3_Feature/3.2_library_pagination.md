
# 3.2 Thư viện Bài học & Phân trang

## 1. Mục đích
Hiển thị danh sách các bài học có trong hệ thống (`data/index.js`) nhưng chia nhỏ ra từng trang để không bị quá tải giao diện.

## 2. Logic Phân trang (Pagination Logic)
Sử dụng công thức toán học để cắt mảng dữ liệu gốc (`lessonsData`).

### Các biến số:
- `currentPage`: Trang hiện tại (State).
- `itemsPerPage`: Số bài trên 1 trang (Cố định = 4).

### Công thức tính toán:

1. **Tìm vị trí cắt:**
```javascript
   const indexOfLastItem = currentPage * itemsPerPage; // Ví dụ trang 1: 1*4 = 4
   const indexOfFirstItem = indexOfLastItem - itemsPerPage; // Ví dụ trang 1: 4-4 = 0

```

2. **Cắt dữ liệu hiển thị:**
```javascript
// Lấy từ index 0 đến 3 (không bao gồm 4)
const currentLessons = lessonsData.slice(indexOfFirstItem, indexOfLastItem);

```


3. **Tính tổng số trang:**
```javascript
// Math.ceil để làm tròn lên (VD: 6 bài / 4 = 1.5 -> 2 trang)
const totalPages = Math.ceil(lessonsData.length / itemsPerPage);

```

## 3. Giao diện (UI Components)

* **Grid Layout:** Sử dụng `grid-cols-1 md:grid-cols-2` để hiển thị 2 bài trên 1 hàng.
* **Pagination Controls:**
* Nút **Prev**: Disable nếu `currentPage === 1`.
* Nút **Next**: Disable nếu `currentPage === totalPages`.
* **Dãy số trang:** Dùng `[...Array(totalPages)]` để tạo mảng số trang động, map ra các nút bấm `1, 2, 3...`.

## 4. Hàm xử lý: `paginate(pageNumber)`

* **Input:** Số trang muốn tới.
* **Action:** `setCurrentPage(pageNumber)`. React sẽ tự động tính lại `currentLessons` và render lại giao diện.
